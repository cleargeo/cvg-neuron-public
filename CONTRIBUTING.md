### Reporting Bugs

1. **Check existing issues** to avoid duplicates
2. **Use the bug report template** when creating a new issue
3. **Provide detailed information:**
   - Clear description of the bug
   - Steps to reproduce
   - Expected vs actual behavior
   - System information (OS, shell version, CVG Neuron version)
   - Screenshots or logs if applicable

### Suggesting Enhancements

1. **Use the feature request template**
2. **Explain the use case** and why it's valuable
3. **Describe your proposed solution**
4. **Consider alternatives** and explain why your approach is preferred

### Submitting Code

1. **Fork the repository**
2. **Create a feature branch:**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes:**
   - Write clean, documented code
   - Follow existing code style
   - Add tests where applicable
   - Update documentation
4. **Commit with clear messages:**
   ```bash
   git commit -m "feat: add new automation script for flood analysis"
   ```
5. **Push to your fork:**
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Submit a pull request** using our PR template

### Shell Scripts
- Use `#!/bin/bash` or `#!/bin/sh` shebang
- Include header comments explaining purpose
- Use meaningful variable names
- Add inline comments for complex logic
- Follow [Google Shell Style Guide](https://google.github.io/styleguide/shellguide.html)

### Python Scripts (if applicable)
- Follow PEP 8 style guidelines
- Include docstrings for functions and classes
- Use type hints where appropriate
- Add unit tests for new functionality

### Documentation
- Use clear, concise language
- Include code examples
- Update README.md if adding new features
- Keep changelog updated
